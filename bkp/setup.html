My Setup

Workflow

I'm not a professional programmer, and I pretty much work alone. So when I'm building chialisp things it's really a process of learning as I build. This means writing a lot of small bits of code just to see how something works. Using my normal approach to programming projects didn't really work here because I ended up with a lot of files and subdirectories scattered everywhere, and it was really hard to get a sense of how all the parts of transactions on the chia network work together.

I have settled on using emacs for all my chialisp work, as it has the flexibility to let your own workflow emerge depending on the tasks you need to do. The major requirement for me was that I wanted to learn chialisp in a literate programming way, so I could write small pieces of chialisp code alongside extensive notes of what I expect it to do and why it's written the way it is. I also want the ability to execute code from the buffer where I write it.

The workflow that emerged from my requirements is to use an org-mode buffer, alongside org-babel for source-code execution, and a few custom functions to handle brun and run of chialisp.

When I want to do something I just write a subheading that concisely says what I want to achieve. For example, "Alice creates a coin that can only be spent alongside an oracle announcement". I can then break that task down into the required chialisp files and python driver code, and I can work on them all at the same time. It is not a perfect setup, but I have found it better than switching between many files and copy-pasting into command line.

And a nice side-effect of org-mode programming is that you can export org files to html without any effort.
